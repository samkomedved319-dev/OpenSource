How to Run, Sync, & Train Locally

Follow our comprehensive step-by-step guides to deploy the CLI, back up vaults, and fine-tune custom local models.

01

How to Run openSource CLI Locally

▼

Make sure you have Node.js (v18+) and Ollama installed. Run the following commands:

# 1. Start your local Ollama server
ollama serve

# 2. Pull the default Source models (or map to Qwen2.5-Coder)
ollama pull qwen2.5-coder:7b

# 3. Launch openSource globally
npm install -g opensource-cli
opensource

# Alternatively, run REPL in a specific project workdir
opensource --workdir ./my-project

Once loaded, type /help in the flat terminal UI to see the list of active slash commands.

02

How to Upload Your CLI Agent Workspace to GitHub

▼

Pushing your project directory to GitHub is simple and allows your local agent to track history:

# 1. Initialize Git in your project root
git init

# 2. Create a standard .gitignore file to exclude heavy files
echo "node_modules/\ndist/\n.env\n.DS_Store" > .gitignore

# 3. Stage and commit your files
git add .
git commit -m "feat: initialize project with openSource CLI"

# 4. Create your remote repository on GitHub, then link and push
git remote add origin https://github.com/your-username/your-repo-name.git
git branch -M main
git push -u origin main

Your openSource CLI can now automatically execute AI commit diffs using the /commit command!

03

How to Upload and Sync Obsidian Vault to GitHub

▼

Keep your Obsidian Knowledge Vault securely synced to GitHub. You have two highly efficient methods:

Method A: Using the Obsidian Git Plugin (Recommended)
  1. Open Obsidian, navigate to Settings > Community Plugins, search for Obsidian Git, and click install.
  2. Ensure Git is installed on your local computer. Initialize a Git repository inside your vault folder: 
    cd "~/Documents/ObsidianVault"
git init
git remote add origin https://github.com/your-username/my-obsidian-vault.git
  3. In Obsidian Git settings, configure **Auto backup interval (minutes)** (e.g., every 30 minutes) and enable **Push on backup**.
  4. The plugin will silently auto-commit and push all your notes in the background!
Method B: Automatic Scripted Backup (Shell Script)

You can create a fast automation script inside your vault directory (e.g., sync_vault.bat on Windows or sync_vault.sh on macOS/Linux):

@echo off
cd "C:\Users\samko\Desktop\OpenSource\Obsidian Vault"
git add .
git commit -m "auto-vault backup: %date% %time%"
git push origin main

You can bind this to Windows Task Scheduler or cron to run hourly, keeping your vault completely backed up.

04

How to Fine-Tune & Train Custom Local Models for Ollama

▼

Because Ollama is strictly an inference runner, it does not support model training directly. Instead, models are fine-tuned using dedicated machine learning libraries (such as Python + Unsloth/Hugging Face) on GPU hardware, compiled into a GGUF file, and then loaded into Ollama. Here is the step-by-step fine-tuning pipeline:

Step 1: Assemble Your Structured Code Dataset

Create a training dataset of your coding patterns in JSONLines format (e.g., dataset.jsonl):

{"instruction": "Create a Fastify schema.", "input": "", "output": "import { z } from 'zod';..."}
{"instruction": "Fix error in list_directory tool.", "input": "depth 0 user profile error", "output": "if (isUserHome(path)) return;..."}
Step 2: Fine-Tune the Base Coding Weights (LoRA)

Use Python scripts with the highly efficient Unsloth or LLaMA-Factory libraries to execute a LoRA fine-tune on a base model (like Qwen2.5-Coder-7B) using an NVIDIA GPU:

from unsloth import FastLanguageModel
import torch

max_seq_length = 2048
model, tokenizer = FastLanguageModel.from_pretrained(
    model_name = "Qwen/Qwen2.5-Coder-7B-Instruct",
    max_seq_length = max_seq_length,
    load_in_4bit = True,
)

# Apply LoRA adapters
model = FastLanguageModel.get_peft_model(
    model,
    r = 16,
    target_modules = ["q_proj", "k_proj", "v_proj", "o_proj", "gate_proj", "up_proj", "down_proj"],
    lora_alpha = 16,
    lora_dropout = 0,
    bias = "none",
)

# Initialize Trainer & run model.fit() on your dataset...
Step 3: Merge Weights and Export as GGUF

Once training finishes, merge the LoRA adapters back into the 16-bit base model and quantize them into a single GGUF weight file:

# Merge and save in 16-bit or 4-bit GGUF format
model.save_pretrained_gguf("custom_source_model", tokenizer, quantization_method = "q8_0")

This generates a compressed custom_source_model-q8_0.gguf file ready for local CPU/GPU execution.

Step 4: Create a Custom Ollama Modelfile

Create a file named Modelfile in your directory referencing your newly compiled weights, and define its systems parameters and prompt patterns:

# Reference the newly compiled GGUF file
FROM ./custom_source_model-q8_0.gguf

# Set inference parameters
PARAMETER temperature 0.3
PARAMETER num_ctx 32768

# Establish agent system prompt
SYSTEM """
You are openSource, an elite local-first AI software engineer.
You prefer clean, functional TypeScript code and autonomous tool-driven solutions.
"""
Step 5: Compile the Model into Ollama

Register the custom model with your local Ollama instance and run it through openSource CLI:

# 1. Compile modelfile inside Ollama
ollama create Source-custom -f Modelfile

# 2. Run the newly trained model in openSource CLI
opensource --model Source-custom

Your local agent is now fully executing the code and patterns you trained it on!
