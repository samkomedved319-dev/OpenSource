Prompt-Based Tool Calling Engine

OpenSource uses a text-based XML tool calling system that works with any local model — no native API tools needed.

How it Works

1
Tool Definitions

Model receives tool descriptions as formatted text in the system prompt

↓
2
Model Responds

Model outputs <tool_call>{JSON}</tool_call> in its text response

↓
3
XML Parsed

Agent extracts the JSON, validates schema, and executes the tool

↓
4
Result Injected

Tool result is injected as a user message — model continues naturally

Why This Matters

  • ✓ No Hangs — Small models don't freeze on native tools API
  • ✓ No Schema Leaking — JSON schema stays in the prompt, not the output
  • ✓ 100% Compatible — Works with any Ollama model, even 3B parameters
  • ✓ Deterministic — XML tags are easy for models to learn and follow
  • ✓ Transparent — You can see exactly what the model is calling

How to Run, Sync, & Train Locally

Follow our comprehensive step-by-step guides to deploy the CLI, back up vaults, and fine-tune custom local models.

01

How to Run OpenSource CLI Locally

▼

Make sure you have Node.js (v20+) and Ollama installed. Run the following commands:

# 1. Start your local Ollama server
ollama serve

# 2. Pull a coding model
ollama pull qwen2.5-coder:7b

# 3. Launch OpenSource globally
npm install -g opensource-cli
opensource

# Or run REPL in a specific project directory
opensource --workdir ./my-project

Once loaded, type /help to see the list of active slash commands.

02

How to Upload Your CLI Workspace to GitHub

▼

Pushing your project directory to GitHub is simple and allows your local agent to track history:

# 1. Initialize Git in your project root
git init

# 2. Create .gitignore
echo "node_modules/\ndist/\n.env\n.DS_Store" > .gitignore

# 3. Stage and commit
git add .
git commit -m "feat: initialize project with OpenSource CLI"

# 4. Link remote and push
git remote add origin https://github.com/your-username/your-repo.git
git branch -M main
git push -u origin main

Your OpenSource CLI can now automatically execute AI commit diffs using the /commit command!

03

How to Sync Obsidian Vault to GitHub

▼

Keep your Obsidian Knowledge Vault securely synced to GitHub:

Method A: Using the Obsidian Git Plugin (Recommended)
  1. Open Obsidian, navigate to Settings > Community Plugins, search for Obsidian Git, and install.
  2. Initialize a Git repository inside your vault folder: 
    cd "~/Documents/ObsidianVault"
git init
git remote add origin https://github.com/your-username/my-obsidian-vault.git
  3. In Obsidian Git settings, configure Auto backup interval (e.g., every 30 minutes) and enable Push on backup.
  4. The plugin will silently auto-commit and push all your notes in the background!
04

How to Fine-Tune Custom Local Models for Ollama

▼

Because Ollama is strictly an inference runner, models are fine-tuned using dedicated ML libraries, compiled into GGUF, and loaded into Ollama.

Step 1: Assemble Your Dataset

Create a training dataset in JSONLines format (dataset.jsonl):

{"instruction": "Create a Fastify schema.", "input": "", "output": "import { z } from 'zod';..."}
{"instruction": "Fix error in list_directory tool.", "input": "depth 0 user profile error", "output": "if (isUserHome(path)) return;..."}
Step 2: Fine-Tune with LoRA (Python + Unsloth)
from unsloth import FastLanguageModel
model, tokenizer = FastLanguageModel.from_pretrained(
    model_name = "Qwen/Qwen2.5-Coder-7B-Instruct",
    max_seq_length = 2048,
    load_in_4bit = True,
)
model = FastLanguageModel.get_peft_model(model, r=16, ...)
Step 3: Export as GGUF
model.save_pretrained_gguf("custom_model", tokenizer, quantization_method = "q8_0")
Step 4: Create Ollama Modelfile
FROM ./custom_model-q8_0.gguf
PARAMETER temperature 0.3
PARAMETER num_ctx 32768
SYSTEM """You are OpenSource, an elite local-first AI software engineer."""
Step 5: Compile and Run
ollama create Source-custom -f Modelfile
opensource --model Source-custom
